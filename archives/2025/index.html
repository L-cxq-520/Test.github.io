变量的本质是2进制

变量的本质是2进制——> 计算机中所有数据的本质都是二进制，是一堆0和1。
为什么是2进制？
数据传递只能通过电信号，只有开和关两种状态。所以就用0和1来表示这两种状态。
计算机中的存储单位最小为bit（位），它只能表示0和1两个数字。
1bit 就是1个数，要么是0要么是1。
为了方便数据表示，出现一个叫byte（字节）的单位，它是由8个bit组成的存储单位。所以我们一般说一个字节为8位。
一个字节的状态举例：
1byte = 0000 0000
1byte = 1111 1111
1字节可以表示2的8次方种状态，即0-255。
有符号数第一个比特充当符号位。当该符号位为0时，表示的是正数，为1时表示负数。

二进制转十进制

从右往左 依次为2的平方叠加
数字为1的2的平方拿出来相加
得出十进制的结果
110101 二进制
1+4+16+32
转换为十进制：53
111101 二进制
1+4+8+16+32
转换为十进制：61
1101011 二进制
1+2+8+32+64
转换为十进制：107

十进制转二进制

把目标值依次减去2的平方 整除到0为止
有次方的为1 没有的则为0
180 十进制
180-4-16-32-128=0
转换成二进制： 10110100
200 十进制
200-128-64-8=0
转换成二进制：11001000
500 十进制
500-256-128-64-32-16-4=0
转换成二进制： 111110100

隐式转换

什么是类型转换

  • 类型转换是指不同变量类型之间的相互转换。隐式转换是指不同类型之间自动转换，通常是从大范围向小范围转换。
  • 隐式转换的基本规则——>不同类型之间自动转换
  • 大范围装小范围

相同的类型之间的转换

有符号

  • long → int → short → sbyte
  • 可以用大范围 装小范围的 类型 （隐式转换）

无符号

  • ulong → uint → ushort → byte
  • 同上（可以用大范围 装小范围的 类型 （隐式转换））

浮点数

  • decimal → double → float
  • decimal这个类型 没有办法用隐式转换的形式 去存储 double和float
  • float 是可以隐式转换成 double

特殊类型

  • bool 、char 、string
  • 他们之间 不存在隐式转换

不同的类型之间的转换

有符号与无符号之间

  • 无符号 不能装负数的

无符号装有符号

  • 有符号的变量 是不能够 隐式转换成 无符号的
  • 都会报错

有符号装无符号

  • 有符号变量 是可以 装 无符号变量的 前提是 范围一定要是涵盖的 存在隐式转换

浮点数与整数（有符号、无符号）之间

  • 浮点数装整数 整形转为浮点数 是存在隐式转换的

  • double ——> float ——> 所有整形（无符号、有符号）

  • decimal ——> 所有整形（无符号、有符号）

  • 整数装浮点数

  • 整数是不能隐式存储 浮点数 因为 整数 不能存小数

特殊类型和其它类型之间

  • bool 类型 没有办法和其它类型 相互隐式转换
  • char 类型 没有办法隐式的存储 其它类型的变量

总结

  • 隐式转换 规则
  • 高精度（大范围）装低精度（小范围）
  • double ——> float ——> 整数（无符号、有符号）——>char
  • decimal ——> 整数（无符号、有符号）——>char
  • string 和 bool 不参与隐式转换规则的
